legacy/training/configs

欢迎来到 legacy/training/configs 目录！这里是整个 AI 项目的 “总设计师办公室” 或 “图纸档案馆”。

如果把训练 AI 比作“造机器人”，那么这个文件夹里放的全是图纸。

1. 这个文件夹主要负责什么功能？

核心功能：定义模型的“出身”和“体格”。

这里的每一个 JSON 文件，都是一份详细的规格说明书。它们不包含任何复杂的逻辑代码（比如怎么计算、怎么反向传播），它们只负责设定参数。

程序在启动训练之前，必须先读这里的文件，弄清楚： * 我们要造的这个脑子有几层楼高？（层数） * 神经元通道有多宽？（隐藏层大小） * 它是用什么架构思考的？（GLA 还是 Transformer） * 它能记多少单词？（词表大小）

简单说：这里决定了模型的上限和基本形态。

2. 这个文件夹下的各个直接文件分别是干什么的？

这里的文件主要分为两类：GLA 家族（主角）和 Transformer（对照组）。

我们可以把它们看作是不同型号的机器人图纸：

🚀 GLA 系列（本项目的主角，新型架构）

这些文件定义了使用 Gated Linear Attention (GLA) 技术的模型。这是一种试图比传统 Transformer 更快、更省显存的新型“大脑构造”。

  • 📄 gla_340M.json (迷你版)
    • 定位：“麻雀型”机器人。
    • 作用：参数量只有 3.4 亿。通常用来快速验证想法、调试代码，或者在显存很小的设备上跑。因为它小，训练和推理都很快。
  • 📄 gla_1B.json (标准版)
    • 定位：“轻量级”机器人。
    • 作用：参数量 10 亿。这是学术界常用的一个小门槛，能看出模型不仅能跑通，而且开始具备一定的智能了。
  • 📄 gla_7B.json (旗舰版)
    • 定位：“重型”机器人。
    • 作用：参数量 70 亿。这是目前开源大模型（如 LLaMA-7B）的标准起步规格。如果我们要证明 GLA 架构真的很强，就必须在这个量级上和别人“掰手腕”。

🐢 Transformer 系列（传统架构，基准线）

  • 📄 transformer_340M.json
    • 定位：“经典款”机器人。
    • 作用：这是标准的 Transformer 架构（和 GPT 一样），大小也是 3.4 亿。
    • 为什么要它？ 它是用来做参照物 (Baseline) 的。当你跑完 gla_340M 的效果，你得跑一下这个 transformer_340M，如果 GLA 跑得比它快、效果比它好，才能证明你的新架构牛逼。

3. 子文件夹的作用是什么？

(注：根据你提供的目录结构，当前层级下没有子文件夹，只有 4 个 JSON 文件。)

如果未来这里出现了子文件夹，通常是为了给不同类型的实验（比如 experiment_A/）或者不同阶段的配置（比如 pretrain/, finetune/）进行归类整理。但目前，这里就是一个平铺的图纸库。

4. 给我一个高层的认知

你可以这样理解这部分代码的作用：

“这里是‘配方’，不是‘厨具’。”

  • 代码 (Code) 是锅碗瓢盆和灶台（在其他文件夹里）。
  • 配置 (Config, 这里的文件) 是菜谱：放多少盐、煮多久、用牛肉还是猪肉。

当你想要改变模型的大小时，你不需要修改任何 Python 代码，你只需要修改或替换这里的一个 JSON 文件。这体现了软件工程中“配置与代码分离”的原则，让做实验变得非常灵活。
