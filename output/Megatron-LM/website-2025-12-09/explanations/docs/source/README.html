docs/source

好，没问题！既然你已经理解了 index.rst 是“首页草稿”，那我们就把视野拉高，看看整个 docs/source 文件夹。

我用“出版社编辑部”的比喻来回答你的三个问题：

1. 这个文件夹主要负责什么功能？

核心功能：存放“说明书的原始手稿”。

如果把整个 Megatron Core 项目比作一台复杂的“超级跑车”： * 外面的代码（.py 文件）是引擎和零件，负责让车跑起来。 * docs/source 文件夹就是手套箱里的《用户手册》。

注意： 这里面的东西绝不会影响跑车的性能，它只负责教驾驶员（也就是你）怎么开车、怎么修车。

2. 各个文件/子文件夹分别是干什么的？

根据 index.rst 的内容，我们可以推断出这个目录下通常会有这三类东西：

  • 📄 index.rst（封面与目录）

    • 角色：这是手册的封面和总目录。
    • 作用：客人进门先看这儿，决定是去学基础，还是去查字典。

  • 📂 user-guide/ 文件夹（新手驾校）

    • 角色：教程区。
    • 内容：这里面的文章是写给人类看的。比如“如何安装”、“十分钟快速上手”、“核心概念讲解”。
    • 比喻：就像教你“左边是刹车，右边是油门”。

  • 📂 api-guide/ 文件夹（维修字典）

    • 角色：字典区。
    • 内容：这里面是冷冰冰的技术细节。列出了所有的函数名、参数类型、返回值。
    • 比喻：就像零件清单，写着“螺丝型号 M5，扭矩 20Nm”。通常只有老司机（开发者）需要查具体参数时才看。

(注：通常这里还会有个 conf.py，那是“排版格式规定”，比如规定字体大小、网站配色，你不用管它。)

3. 高层认知：一句话理解这部分代码

请把这个文件夹看作：项目的“宣发部门”。

  • 别去试图“运行”它：这里的代码跑不通任何 AI 模型。
  • 它是给工具看的：有一个叫 Sphinx 的“打印机”程序，会读取这个文件夹里的所有文字，然后“打印”成你平时在浏览器里看到的那个漂亮的文档网站。

总结：这里是写文档的地方，不是写代码的地方。
