✅ Task 2：搞懂“优化器”和它的参数（基础篇）

目标： 理解文中提到的 Optimizer（优化器）、Learning Rate（学习率）和 Weight Decay（权重衰减）。

想象你在训练一个AI模型，就像在教一个小孩子学骑自行车。

  1. Optimizer (优化器)：
    • 这就是你（教练）。你的任务是根据小孩子骑得歪不歪，去调整他的姿势，让他骑得更稳。
  2. Learning Rate (学习率)：
    • 这是你纠正动作的幅度。
    • 学习率太大： 你猛推一把，孩子直接摔倒了（模型学崩了）。
    • 学习率太小： 你轻轻碰一下，孩子根本感觉不到变化，学得慢死（模型收敛太慢）。
  3. Weight Decay (权重衰减)：
    • 这是为了防止孩子“耍小聪明”（比如只死记硬背某一条路怎么骑，换条路就不会了）。
    • 在AI里，这是一种“惩罚”机制，让模型保持简单，不要过度拟合（Overfitting）。

✅ Task 3：搞懂“Scheduler”（核心逻辑篇）

目标： 理解标题里的 Scheduler（调度器）是干嘛的。

既然我们要教孩子（训练模型），“学习率”应该是一成不变的吗？ 当然不是！

  • 刚开始学： 孩子啥都不会，你要大声指导，动作幅度大一点让他以此为准（学习率较大）。
  • 学到中间： 孩子大概会了，你要稳住节奏。
  • 最后阶段： 孩子骑得挺好了，你只需要微调，轻轻扶一下把手就行（学习率要变小）。

这就是 Scheduler（调度器）的作用： 它是一个自动的时间表。它负责在训练过程中，根据训练的进度（比如练了多少轮），动态地改变学习率（Learning Rate）和权重衰减（Weight Decay）。
