2. 这个文件夹下的各个文件/子文件夹分别是干什么的？

通常在这个目录下，最核心的就是你刚才看到的那个 YAML 文件，有时候可能还会伴随一两个辅助文件。我们可以这样理解：

  • 📂 文件夹本身 (bert-nemo_340m_...)：

    • 这是“病历袋”或者“档案袋”。
    • 那串长长的名字（bert-nemo_340m_mr_mbs2_...）就是标签，写满了这次测试的所有特征。测试脚本通常只看这个文件夹名字，就能知道大概要干什么。

  • 📄 model_config.yaml (核心文件)：

    • 这是“操作手册”或“处方单”。
    • 它详细规定了具体的参数：用什么药（模型）、剂量多少（Batch Size）、怎么分工（并行策略）。这是必须要有的，没有它，机器就不知道怎么跑。

  • (如果存在) *.sh (脚本文件)：

    • 这是“启动按钮”。
    • 有时候为了方便，会放一个脚本直接运行这个配置。但在自动化测试系统中，通常不需要这个，系统会自动读取 YAML 来跑。
