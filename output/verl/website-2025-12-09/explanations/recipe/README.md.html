为了让你听懂，我把阅读这份文档拆解成 4个具体的任务（Task），你只需要按照这个顺序一步步来，就能完全理解它在讲什么。

📋 你的学习任务清单 (To-Do List)

✅ Task 1：看懂最重要的“搬家通知”

原文核心：

Update 2025/11/25: recipes have been moved to a new repository: [verl-recipe]...

解读： 这是这份文档里最重要的一句话。 * 发生了什么？ 这里的代码（Recipes）已经“搬家”了。 * 去哪里了？ 搬到了一个新的仓库叫 verl-recipe。 * 结论： 如果你想找最新的、能用的标准代码，不要在这个文件夹里找，要去那个新链接。

✅ Task 2：搞清楚谁是“锅”，谁是“菜谱”

原文核心：

verl is designed to be a modular... framework... Recipe is expected to import verl as a library...

解读： 这里解释了两个核心概念的关系：verl 和 Recipe。我们可以用做饭打比方：

  1. verl (锅具/工具箱)：

    • 它是一个框架。
    • 它是用来给 AI 模型做“考前突击辅导”的（专业术语叫 Post-training，包括 SFT 监督微调 和 RL 强化学习）。
    • 它提供了基础的工具和功能。

  2. Recipe (菜谱/具体做法)：

    • 它是具体的训练脚本或流程。
    • 它引用 verl 这个工具箱，来搭建具体的训练流水线。
    • 意思就是： 如果你想训练一个像 DeepSeek 那样的模型，你需要写一个 Recipe（菜谱），在这个菜谱里调用 verl（锅具）来炒菜。

✅ Task 3：了解留下的“实验品”

原文核心：

There's still some incubation recipes kept here... - fully_async_policy... - vla...

解读： 既然大家都搬家了，为什么这里还剩下几个文件？ * Incubation (孵化中)： 这些是还在实验阶段、不太成熟的功能，还没准备好正式搬到新家，所以暂时留在这里“孵化”。 * 留下了啥？ * fully_async_policy：一种追求极致速度的训练方法（完全异步）。 * vla：专门给机器人或者视觉模型用的训练方法。

原文核心：

Awesome work using verl
