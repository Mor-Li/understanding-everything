目标：这本书里不仅有文字，还有数学公式、代码块，甚至需要直接从代码里提取注释。我们需要安装相应的“插件”来处理这些内容。

    myst_enable_extensions = [ "dollarmath", # 支持用 $ 符号写数学公式 "amsmath", ] ``` * 解读： * 这就好比你告诉印刷厂：“我要支持 Markdown 格式，我要支持数学公式渲染，我还要你能自动读取我的代码注释。”

✅ Task 3: 设定“语言规范” (文件格式与风格)

目标：规定好源文件的后缀名，以及代码注释的风格，防止乱码或排版错误。

  • 对应代码： ```python # 告诉工具，.rst 和 .md 结尾的文件都是文档源文件 source_suffix = { ".rst": "restructuredtext", ".md": "markdown", }

    napoleon_google_docstring = True napoleon_numpy_docstring = False `` * **解读**： * 这就像规定：“投稿请用.md` 格式，写注释请遵循 Google 的格式规范。”
